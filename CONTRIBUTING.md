# Contributing to UnleashedSMP

Thanks for your interest in contributing!

## Guidelines

- Follow clean architecture and SOLID principles.
- Keep every configurable value in a config file (no hardcoding).
- Add Javadoc to public APIs.
- Use MiniMessage for all user-facing text.
- Ensure `mvn clean package` succeeds and tests pass.

## Pull Requests

1. Fork the repository.
2. Create a feature branch.
3. Make your change with tests.
4. Open a PR describing the change and its motivation.

## Code Style

Google Java Style, Java 21, KISS/DRY, meaningful naming.
